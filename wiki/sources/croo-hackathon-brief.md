# Source Note: CROO AI Agent Hackathon Brief

Source: `docs/CROO AI AGENT Hackathon.docx`

Date ingested: 2026-06-26

## What The Challenge Asks For

The challenge asks participants to build paid, callable AI agents on the CROO agent commerce layer.

The local brief frames CROO as decentralized commerce infrastructure for the AI agent economy. CAP, the CROO Agent Protocol, is described as a permissionless A2A standard that lets agents discover, hire, and pay other agents on-chain. The CROO Agent Store is described as a marketplace where agents are listed with wallets, services, pricing, and transaction flows.

## Submission Requirements

Every build must satisfy five requirements:

- listed on CROO Agent Store
- integrated with CAP so the agent is callable and settles on-chain
- open source with a permissive license such as MIT or Apache 2.0
- demo plus README, including setup instructions, SDK methods used, and integration notes
- BUIDL filed on DoraHacks with required fields complete

## Tracks

The local brief lists these tracks:

- Research & Intelligence Agents
- Data & Verification Agents
- Creator & Content Ops Agents
- DeFi / On-chain Ops Agents
- Developer Tooling Agents
- Open - Any A2A Agents

Multiple-track submissions are allowed, with a maximum of two tracks per BUIDL.

## Implications For This Project

This is not a typical Kaggle tabular competition. There is no local evidence of a leaderboard metric or training dataset in the workspace. The strongest submission must therefore optimize for:

- working product demo
- real or clearly simulated CAP lifecycle
- credible A2A composability
- high-quality open-source repo
- clear business model and pricing
- reproducible evidence that the agent works
- judge-friendly README, demo video, and writeup
