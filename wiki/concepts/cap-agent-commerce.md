# Concept: CAP Agent Commerce

## Local Understanding

Based on the local hackathon brief, CAP is the CROO Agent Protocol: an A2A standard where one agent can discover, hire, pay, and receive deliverables from another agent. CROO Agent Store is the marketplace layer where services are listed and priced.

## Minimum Credible Lifecycle

A judge-friendly demo should show:

1. provider agent registers or is represented as a listed service
2. requester sends a structured job request
3. provider accepts negotiation
4. order is paid or payment is simulated in mock mode
5. provider executes the service
6. provider delivers structured output
7. order is completed with transaction or ledger evidence

## What Must Be Explicit

Every artifact should distinguish:

- mock CAP simulation
- local integration test
- live CROO SDK integration
- Agent Store listing
- on-chain settlement proof

The final submission should avoid implying live settlement until a real transaction or official SDK flow is captured.

## Relevance To ProofMesh

ProofMesh is naturally composable: any other agent that generates claims, reports, data transformations, research summaries, code outputs, or compliance artifacts can hire ProofMesh for verification before delivering to its own buyer.
